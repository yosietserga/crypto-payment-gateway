API Documentation

Reference for integrating with the Crypto Payment Gateway API

Getting Started with the API

The Crypto Payment Gateway API allows you to accept cryptocurrency payments and send payouts programmatically. Our RESTful API provides a simple and secure way to integrate cryptocurrency transactions into your applications.

Base URL
http://localhost:3000/api/v1
API Requirements
  • All requests must use HTTPS
  • Request and response bodies are formatted in JSON
  • Authentication is required for all endpoints except public endpoints
Rate Limits

The API enforces rate limits to ensure fair usage. Current limits are:

  • Authentication endpoints: 10 requests per minute
  • Payment and payout operations: 60 requests per minute
  • Read operations: 120 requests per minute

Authentication

The API uses API keys and HMAC signatures for authentication.

API Keys

You can generate API keys in the API Keys section of your dashboard.

Headers

Include the following headers with each API request:

Header Description
X-API-KEY Your API key
X-TIMESTAMP Current Unix timestamp in milliseconds
X-SIGNATURE HMAC-SHA256 signature of the request
Generating Signatures

Generate the signature by creating an HMAC-SHA256 hash of the request data, signed with your API secret.

signature = HMAC-SHA256(timestamp + method + path + body, apiSecret)

Payments API

Endpoints for generating payment addresses and managing payments.

Create Payment

Generate a new payment address for receiving funds.

POST /merchant/payment-addresses

{
  "currency": "USDT",
  "expectedAmount": "100.00",
  "callbackUrl": "https://example.com/webhook",
  "metadata": {
    "orderId": "12345",
    "customer": "example@email.com"
  }
}
Get Payment

Retrieve information about a payment.

GET /merchant/payment-addresses/{addressId}
List Payments

Get a list of payments with pagination.

GET /merchant/payment-addresses?limit=20&page=1&status=ACTIVE

Payouts API

Endpoints for sending cryptocurrency payouts.

Create Payout

Initiate a payout to a cryptocurrency address.

POST /merchant/payouts

{
  "amount": "50.00",
  "currency": "USDT",
  "recipientAddress": "0x1234567890abcdef1234567890abcdef12345678",
  "description": "Monthly payment",
  "metadata": {
    "invoiceId": "INV-001"
  },
  "callbackUrl": "https://example.com/payout-webhook"
}
Get Payout

Retrieve information about a payout.

GET /transactions/{transactionId}
List Payouts

Get a list of payouts with pagination.

GET /transactions?limit=20&page=1&type=PAYOUT&status=COMPLETED

Webhooks

Configure and manage webhooks for real-time notifications.

Event Types
Event Description
payment.created A new payment has been created
payment.pending Payment has been detected but not confirmed
payment.completed Payment has been confirmed
payment.expired Payment has expired
payout.created A new payout has been created
payout.processing Payout is being processed
payout.completed Payout has been completed
payout.failed Payout has failed
Webhook Payload

Example webhook payload:

{
  "id": "evt_123456789",
  "type": "payment.completed",
  "created": 1651234567890,
  "data": {
    "id": "pay_987654321",
    "amount": "100.00",
    "currency": "USDT",
    "status": "completed",
    "address": "0xabcdef1234567890abcdef1234567890abcdef12",
    "txHash": "0x9876543210abcdef9876543210abcdef9876543210abcdef9876543210abcdef",
    "metadata": {
      "orderId": "12345"
    }
  }
}
Verifying Webhooks

Verify webhook authenticity by checking the signature:

// The signature is included in the X-Signature header
// Verify by creating an HMAC-SHA256 hash of the request body
// using your webhook secret

const signature = HMAC-SHA256(requestBody, webhookSecret)
if (signature === request.headers['X-Signature']) {
  // Webhook is authentic
}
